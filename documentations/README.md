# documentations

This directory is for longer-lived technical references.

Keep this folder for:
- architecture notes that are still useful after the current cleanup cycle
- CAN references and DBC files
- vendor manuals that are worth versioning with the workspace
- drawings and exported reference assets

Use this folder for reference material, not for daily work notes.

Current entry points:
- [CURRENT_ARCHITECTURE.md](./CURRENT_ARCHITECTURE.md)
- [README_CAN.md](./README_CAN.md)
- `MTT_CAN_v1_1_simple.dbc`
- `mttDriver.dbc`

If a file is mainly about how to run the workspace or how to launch a demo, it belongs in `doc/` or in the relevant demo README instead.
